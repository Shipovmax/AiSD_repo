# Дополнительный список задач для подготовки к экзамену по АиСД (2026)

## 1. Объектно-ориентированное программирование (ООП)

- [ ] **Задача 1 (Легкая)**: Опишите класс `Product`, заданный названием, ценой и количеством. Включите в описание
  класса методы: вывода информации о товаре на экран, проверки, есть ли товар в наличии (количество больше 0), и
  свойство, позволяющее установить категорию товара.
- [ ] **Задача 2 (Ниже средней)**: Создайте класс `Студент` (имя, фамилия, возраст, список оценок). Реализуйте методы
  добавления оценка, вычисления среднего балла и вывод информации. Используйте магический метод `__len__` для получения
  количества оценок студента.
- [ ] **Задача 3 (Средняя)**: Создайте родительский класс `ТРАНСПОРТ` (информация, базовая грузоподъемность). Реализуйте
  дочерние классы:
    - `АВТОМОБИЛЬ` (наследует базовые параметры)
    - `МОТОЦИКЛ` (без коляски, жестко задает $\text{грузоподъемность} = 0$)
    - `ГРУЗОВИК` (при наличии прицепа динамически увеличивает $\text{грузоподъемность} \times 2$)
      Организуйте коллекцию объектов и реализуйте функцию поиска транспортных средств по строго заданному диапазону
      грузоподъемности.
- [ ] **Задача 4 (Выше средней)**: Создайте класс `Квадрат`. Координаты верхнего левого угла и длина стороны должны быть
  приватными атрибутами. Реализуйте метод, который "рисует" закрашенный квадрат в консоли (символами, например, `*`), а
  также методы изменения размера и валидации координат при попытке их обновления.
- [ ] **Задача 5 (Сложная)**: Создайте класс `Дом` (этажность, подъезды, район, список работающих на объекте
  сотрудников) и класс `Рабочие` (компания, квалификация). Каждый рабочий может быть закреплен за объектом в
  определенный промежуток времени. Напишите метод, принимающий на вход конкретного рабочего и целевой год (формат даты
  `01.01.2022`), который определяет, в скольких постройках данный рабочий был задействован в течение этого года.

## 2. Декораторы

- [ ] **Задача 1 (Легкая)**: Реализуйте декоратор, который проверяет возвращаемое функцией число: если оно четное, то
  возводит его в квадрат, если нечетное — в куб.
- [ ] **Задача 2 (Ниже средней)**: Создайте функцию `three_args(a, b, c)`, принимающую строго три ключевых параметра.
  Напишите декоратор, который перехватывает выполнение и выводит `0`, если хотя бы один из переданных параметров равен
  `None`, не вызывая саму функцию.
- [ ] **Задача 3 (Средняя)**: Реализуйте параметризованный декоратор `dec(a, b)`. Декоратор должен анализировать
  результат математической функции: если возвращаемая сумма положительна, он увеличивает итоговый результат на величину
  `a`, а если отрицательна — уменьшает на величину `b`.
- [ ] **Задача 4 (Выше средней)**: Реализуйте параметризованный декоратор `decor(a)`. Он должен применять изменения к
  возвращаемому функцией словарю: все строковые значения, чьи ключи являются числовыми и кратны параметру `a`, должны
  быть приведены к верхнему регистру (`UPPERCASE`).
- [ ] **Задача 5 (Сложная)**: Создайте параметризованный декоратор `tol(dlina, fill)`. Он должен трансформировать
  результат работы декорируемой функции в словарь, состоящий строго из `dlina` элементов. Если функция возвращает меньше
  элементов, декоратор заполняет недостающие позиции значением `fill`. Если функция возвращает больше элементов —
  декоратор жестко "обрезает хвост" до заданной длины.

## 3. Стек

- [ ] **Задача 1 (Легкая)**: Реализуйте класс `Стек` на базе стандартного списка (`list`). Напишите метод, который
  меняет местами самый первый (нижний) и самый последний (верхний) элементы в стеке, не разрушая промежуточную
  структуру.
- [ ] **Задача 2 (Ниже средней)**: Реализуйте класс `Стек`. Напишите функцию, которая принимает объект стека и удаляет
  из него каждый второй элемент (начиная сверху).
- [ ] **Задача 3 (Средняя)**: Реализуйте класс `Стек`. Напишите алгоритм, который вычисляет среднее арифметическое всех
  числовых элементов в стеке, а затем подсчитывает точное количество элементов, значения которых строго больше этого
  среднего значения.
- [ ] **Задача 4 (Выше средней)**: Реализуйте класс `Стек`. Напишите функцию проверки содержимого: она должна
  определить, является ли последовательность элементов в стеке строго упорядоченной по возрастанию (от нижнего элемента
  к верхнему). Удалите из стека все элементы, которые не являются квадратами целых чисел.
- [ ] **Задача 5 (Сложная)**: Создайте специализированный класс `Стек`, принимающий при инициализации кастомное правило
  фильтрации (например, лямбда-функцию или ограничение на длину строк). Стек должен принимать элементы через метод
  `push()` только в том случае, если они удовлетворяют заданному правилу. Дополнительно реализуйте метод, который
  считывает три нижних элемента структуры и меняет местами верхний и нижний из этой тройки.

## 4. Однонаправленные связные списки

- [ ] **Задача 1 (Легкая)**: Реализуйте структуру данных однонаправленного связного списка и напишите функцию добавления
  нового элемента в начало (слева) списка.
- [ ] **Задача 2 (Ниже средней)**: Реализуйте метод добавления нового узла в конец (справа) однонаправленного связного
  списка, корректно обрабатывая случай, когда список изначально пуст.
- [ ] **Задача 3 (Средняя)**: Напишите функцию для однонаправленного связного списка, которая осуществляет поиск
  минимального значения среди всех узлов, а также функцию подсчета суммы элементов, значения которых кратны 3.
- [ ] **Задача 4 (Выше средней)**: Реализуйте функцию вставки нового узла в однонаправленный связный список. Функция
  должна поддерживать два режима работы: вставка после узла с заданным порядковым номером (индексом) или вставка после
  узла с определенным целевым значением.
- [ ] **Задача 5 (Сложная)**: Реализуйте функцию поиска и удаления узла по заданному значению в однонаправленном связном
  списке. Необходимо корректно перестроить ссылки соседних узлов, обработать случаи удаления головного (первого)
  элемента, единственного элемента списка, а также ситуацию, когда элемент не найден.

## 5. Массивы

- [ ] **Задача 1 (Легкая)**: Дан массив целых чисел. Напишите программу, которая находит в нем количество пар соседних
  элементов, имеющих разные знаки (один положительный, другой отрицательный).
- [ ] **Задача 2 (Ниже средней)**: Даны два целочисленных массива произвольной длины. Напишите эффективный алгоритм,
  который формирует третий массив, состоящий только из общих элементов, присутствующих в обоих исходных массивах (
  пересечение множеств).
- [ ] **Задача 3 (Средняя)**: Дан массив целых чисел. Рассчитайте остаток от деления на 7 для каждого числа. Напишите
  программу, которая находит произведение всех вычисленных остатков, которые строго больше 4.
- [ ] **Задача 4 (Выше средней)**: Дан массив целых чисел (элементы генерируются в диапазоне от -30 до 30). Напишите
  алгоритм проверки: если в массиве присутствует хотя бы один отрицательный элемент, массив должен быть отсортирован по
  возрастанию. Если отрицательных элементов нет — отсортируйте его по убыванию.
- [ ] **Задача 5 (Сложная)**: Дан массив целых чисел (диапазон от -20 до 20). Реализуйте комплексную сортировку по
  условию: если модуль суммы всех отрицательных элементов массива строго больше суммы всех положительных
  элементов ($\left| \sum \text{отрицательных} \right| > \sum \text{положительных}$), отсортируйте массив по
  возрастанию. В противном случае выполните сортировку по убыванию.

## 6. Функции высшего порядка и Лямбда-выражения

- [ ] **Задача 1 (Легкая)**: Используя исключительно лямбда-функции и встроенные инструменты, реализуйте:
    - а) Вычисление среднего арифметического числового списка.
    - б) Поиск квадратов всех чисел в списке.
- [ ] **Задача 2 (Ниже средней)**: Используя лямбда-функции, напишите алгоритмы для:
    - а) Сортировки списка строк по их последней букве.
    - б) Сортировки списка кортежей по значению их первого элемента.
- [ ] **Задача 3 (Средняя)**: Дан список целых чисел. С помощью комбинации функций `map`, `filter` и `reduce` найдите
  произведение всех отрицательных чисел, а также сумму элементов, которые строго меньше нуля.
- [ ] **Задача 4 (Выше средней)**: Дан список слов. С помощью функций высшего порядка (`map`, `filter`, `reduce`)
  рассчитайте остаток от деления длины каждого слова на 7. Затем найдите итоговое произведение тех остатков, которые
  оказались строго больше 4.
- [ ] **Задача 5 (Сложная)**: Дано развернутое текстовое предложение. С помощью связки функций `map`, `filter` и
  `reduce` обработайте его следующим образом: удалите первую и последнюю букву у абсолютно каждого слова, а затем
  склейте в одну финальную строку через пробел только те модифицированные слова, чья длина строго превышает 5 символов.

## 7. Сортировка

- [ ] **Задача 1 (Легкая)**: Реализуйте классический алгоритм сортировки методом пузырька (Bubble Sort) для
  упорядочивания списка целых чисел по возрастанию.
- [ ] **Задача 2 (Ниже средней)**: Дан список объектов пользователей, где у каждого пользователя есть строковое имя и
  числовой возраст. Реализуйте сортировку этого списка по возрасту, используя метод выбора (Selection Sort).
- [ ] **Задача 3 (Средняя)**: Модифицируйте ранее созданный класс `Автомобиль` (содержащий марку, модель и год выпуска).
  Добавьте в него кастомный метод класса, который принимает список объектов-автомобилей и сортирует его по алфавитному
  порядку марки (от A до Z), используя оптимизированный алгоритм пузырька.
- [ ] **Задача 4 (Выше средней)**: Реализуйте алгоритм сортировки методом Шелла (Shell Sort) с использованием
  классической последовательности шагов. Примените его для сортировки списка городов (объектов или словарей) по уровню
  их населения от меньшего к большему.
- [ ] **Задача 5 (Сложная)**: Напишите программу, которая генерирует список сложных структур (например, кортежи вида (
  название_товара, цена, рейтинг)). Реализуйте устойчивую сортировку методом Шелла, которая способна динамически
  переключать критерий сортировки (либо по цене, либо по рейтингу) в зависимости от переданного в функцию флага,
  сохраняя при этом исходный порядок элементов с одинаковыми значениями.

## 8. Бинарные деревья поиска (BST)

- [ ] **Задача 1 (Легкая)**: Реализуйте структуру узла бинарного дерева поиска `Node` (значение, ссылки на левого и
  правого потомков) и сам класс дерева `BinarySearchTree`. Напишите рекурсивный метод для вставки нового числового
  ключа.
- [ ] **Задача 2 (Ниже средней)**: Напишите методы обхода бинарного дерева поиска тремя классическими способами:
  прямым (Pre-order), симметричным (In-order) и обратным (Post-order). Проверьте свойство BST: выводит ли симметричный
  обход элементы в строго отсортированном порядке.
- [ ] **Задача 3 (Средняя)**: Реализуйте функцию поиска элемента в BST. Напишите два варианта: рекурсивный и
  итеративный (без использования рекурсии, через цикл `while`). Сравните их логику. Дополнительно напишите методы поиска
  минимального и максимального ключей в дереве.
- [ ] **Задача 4 (Выше средней)**: Реализуйте метод `height()`, возвращающий высоту бинарного дерева поиска. Напишите
  функцию, которая проверяет, является ли дерево сбалансированным (высоты левого и правого поддеревьев каждого узла
  различаются не более чем на 1).
- [ ] **Задача 5 (Сложная)**: Реализуйте полноценный алгоритм удаления узла (`delete`) из бинарного дерева поиска.
  Алгоритм должен корректно обрабатывать все три случая:
    - Узел является листом (нет потомков).
    - Узел имеет только одного потомка (левого или правого).
    - Узел имеет двух потомков (с поиском наименьшего элемента в правом поддереве для замены).
